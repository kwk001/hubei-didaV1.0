# 视频教材入库管理页面 - 开发复盘与经验总结 (Final Version)

## 1. 项目背景
本任务旨在构建一个服务于教师端的“视频教材入库管理”页面。项目经历了从基础 CRUD 到多 Tab 分类管理，再到细节交互优化（如视频预览、布局修复）的完整迭代过程。最终成果是一个符合低代码规范、数据仿真度高、交互稳健的生产级页面。

## 2. 核心成果
- **标准组件化架构**：成功集成了 `@alifd/next` 的 `Tab` 组件与 `KaiwuFlexLayout`、`KaiwuFlexTable2`，实现了“我的视频教材”与“公用视频教材”的物理隔离与逻辑联动。
- **高仿真数据环境**：定制了“高职院校信息技术专业”的 Mock 数据（如 Java、Python 课程），并采用“2025级”等真实年级格式，极大提升了演示效果。
- **稳健的交互体验**：解决了 Tab 切换下的组件渲染延迟问题，并修复了新增按钮在特定布局配置下的不可见缺陷。
- **原生级预览功能**：通过 `KaiwuJsx` + 原生 HTML 方案，绕过了低代码组件的渲染异常，实现了零依赖的图片/视频预览弹窗。

## 3. 关键问题与解决方案（避坑指南）

### 3.1 布局塌陷与按钮消失
**问题现象**：在 Tab 内部使用 `KaiwuFlexLayout` 时，配置在 `headerBtnGroupLeft` 的“新增教材”按钮无法显示。
**根因分析**：
1. **懒加载干扰**：`Tab` 组件默认懒加载未激活的标签页，导致内部布局组件初始化时无法正确计算宽高。
2. **栅格配置错误**：`KaiwuFlexLayout` 的 `labelColSpan` 设置为 `-1` 时，在某些嵌套环境下会导致头部区域高度塌陷。
**解决方案**：
- 在 `Tab` 组件上开启 `unmountInactiveTabs={true}`，确保每次切换都触发生命周期重置。
- 将 `labelColSpan` 修正为正值（如 `6`），配合 `wrapperColSpan={18}`，显式撑开布局空间。

### 3.2 组件依赖缺失导致的白屏
**问题现象**：点击预览时，使用 `window.LowcodeEngineComponents.KaiwuFlexDialog2` 导致页面报错或无响应。
**根因分析**：低代码运行时的全局对象 `window.LowcodeEngineComponents` 可能未暴露特定组件，或者组件版本不兼容。
**解决方案**：
- **降级策略**：在 `renderPreview` 方法中放弃对平台组件的依赖。
- **原生实现**：直接使用 React/JSX 渲染原生的 `div` 遮罩层、`img` 和 `video` 标签，通过 `fixed` 定位覆盖全屏。这不仅解决了报错，还减少了对平台版本的耦合。

### 3.3 数据源的灵活适配
**问题现象**：默认 Mock 数据（K12 语文/数学）不符合高职院校演示场景。
**解决方案**：
- 在 `index.jsx` 的 `getMockData` 中重构数据生成逻辑，注入具有行业特征的数据（Java 环境搭建、Docker 部署等）。
- 同步更新 `schema.json` 中的搜索下拉选项，确保“查询条件”与“列表数据”的一致性。

## 4. 技术沉淀
1. **组件组合原则**：复杂页面（如 Tab 页）优先采用“物理分离”策略，即每个 Tab Item 内部独立维护一套 Layout + Table，虽然代码冗余，但能规避绝大多数状态共享导致的 Bug。
2. **逃生舱机制**：当标准低代码组件（如弹窗、播放器）无法满足需求或运行异常时，`KaiwuJsx` 配合原生 HTML/CSS 是最可靠的“逃生舱”。
3. **调试技巧**：遇到布局问题（如按钮不显示），优先检查父容器的 `display` 属性和子组件的栅格配置（ColSpan），而非盲目修改样式。
