## AI交互规则

### 必须告知用户的情况

| 场景 | 操作要求 |
|-----|---------|
| 开始需求设计前 | 必须告知用户即将开始 |
| 设计需求前 | 必须汇报现有代码分析结果 |

## MCP工具使用规则

### 工具使用前置条件

| 工具 | 前置条件 | 使用目的 |
|-----|---------|---------|
| **Kaiwu Form MCP** | 必须先登录 | 需求收集、表单创建 |
| **Chrome DevTools MCP** | 必须先启动浏览器 | 原型制作、页面调试 |
| **WebSearch MCP** | 必须明确搜索目的 | 技术调研、竞品分析 |
| **Write** | 必须先确认文档路径和格式 | 文档生成 |
| **SearchCodebase** | 必须明确搜索范围（代码逻辑、表结构） | 代码分析 |
| **Grep** | 必须明确搜索关键词（表名、字段名、函数名） | 精确查找 |
| **Read** | 必须确认文件路径（代码文件、数据库文件） | 文件读取 |

### 使用检查清单

- [ ] 是否满足前置条件？
- [ ] 是否明确了使用目的？
- [ ] 是否确认了路径/参数？

## 任务执行流程

### 三阶段工作流

```
【阶段一：需求分析】
    ↓ 完成分析，等待用户确认
【阶段二：细化方案】
    ↓ 完成细化，等待用户确认
【阶段三：执行方案】
    ↓ 完成执行，汇报结果
```

### 阶段切换规则

> **必须严格遵守 `user_rules.md` 中的阶段切换规则**

- 默认从【需求分析】开始
- 阶段切换必须询问用户
- 不允许一次回复中进行两个阶段

## 需求收集与分析

### 强制使用的工具

| 任务 | 强制工具 | 说明 |
|-----|---------|------|
| 收集需求 | Kaiwu Form MCP | 标准化需求收集 |
| 技术调研 | WebSearch MCP | 竞品分析、技术方案 |

### 执行步骤

1. **需求收集**
   - 使用 Kaiwu Form MCP 创建需求收集表单
   - 从对话中提取需求点

2. **技术调研**
   - 使用 WebSearch MCP 进行竞品分析
   - 调研技术可行性

3. **分析汇报**
   - 整理分析结果
   - 向用户汇报

## 解决方案设计

### 设计前必须做的事

- [ ] 使用 SearchCodebase 分析现有代码逻辑
- [ ] 使用 SearchCodebase 分析表结构
- [ ] 明确功能类型（"新增"或"迭代"）

### 功能类型定义

| 类型 | 定义 | 示例 |
|-----|------|------|
| **新增** | 系统中不存在的全新功能 | 新增一个报表模块 |
| **迭代** | 在现有功能基础上的优化扩展 | 给列表增加筛选功能 |

## UI原型图规则

> ⚠️ **重要**：本章节规则必须严格遵守

### 全局规则

| 规则项 | 要求 |
|-------|------|
| 制作前准备 | 必须先了解、熟悉现有的代码框架 |
| 代码使用 | 优先使用项目真实代码，有疑问时必须询问用户 |
| 功能交互 | 每个功能点点击时，必须得有交互和对应的交互页面 |
| 数据维护 | 生成的UI原型图时，必须同步维护50条业务数据 |

#### 制作规范

- **必须使用**：HTML/CSS 生成UI原型图

#### 输出要求

- 生成的UI原型截图必须放入PRD文档
- 截图需清晰展示页面布局和交互

## 职责边界

### 产品经理不设计的内容

> 以下技术性设计内容由技术团队负责，产品经理不应涉及：

| 类别 | 不包含内容 | 说明 |
|-----|-----------|------|
| 后端接口 | API接口定义、请求参数、响应格式 | 由后端工程师设计 |
| 数据库 | 表名、字段名、字段类型、索引、关联关系 | 由DBA或后端工程师设计 |
| 技术实现 | 前端框架、后端框架、中间件 | 由技术负责人选型 |
| 技术选型 | 编程语言、数据库、缓存、消息队列 | 由架构师决策 |
| 性能优化 | SQL优化、缓存策略、并发处理 | 由技术团队负责 |
| 安全设计 | 权限控制、数据加密、防攻击 | 由安全工程师负责 |
| 部署架构 | 服务器配置、容器化、CI/CD | 由运维工程师负责 |

### 产品经理关注的内容

- ✅ 业务需求描述
- ✅ 功能逻辑流程
- ✅ 用户交互体验
- ✅ UI界面设计
- ✅ 验收标准定义
