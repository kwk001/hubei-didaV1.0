# Agent Skills 规范

技能是一个包含指令、脚本和资源的文件夹,代理可以动态发现和加载这些内容,以更好地执行特定任务。为了使文件夹被识别为技能,它必须包含一个 `SKILL.md` 文件。

# 技能文件夹布局

一个最小的技能文件夹如下所示:

```
my-skill/
  - SKILL.md
```

更复杂的技能可以根据需要添加其他目录和文件。

# SKILL.md 文件

技能的"入口点"是 `SKILL.md` 文件。这是唯一必须存在的文件。该文件必须以 YAML 前置元数据开始,然后是常规的 Markdown 内容。

## YAML 前置元数据

YAML 前置元数据有 2 个必需属性:

- `name`
  - 连字符格式的技能名称
  - 仅限小写 Unicode 字母数字 + 连字符
  - 必须与包含 SKILL.md 的目录名称匹配
- `name_chn`
  - 中文名称
  - 可以为空
- `description`
  - 技能功能和 Claude 何时应该使用它的描述

有 3 个可选属性:

- `license`
  - 应用于该技能的许可证
  - 我们建议保持简短(许可证名称或捆绑许可证文件的名称)
- `allowed-tools`
  - 预先批准运行的工具列表
  - 目前仅在 Claude Code 中支持
- `metadata`
  - 从字符串键到字符串值的映射
  - 客户端可以使用此项存储 Agent Skills Spec 未定义的其他属性
  - 我们建议使您的键名称具有合理的唯一性以避免意外冲突

## Markdown 正文

Markdown 正文没有限制。
