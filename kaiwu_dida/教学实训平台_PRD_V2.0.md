## 一、产品概述

### 1.1 产品定位
**产教融合一体化教学实训管理平台** —— 打通教学管理系统（LMS）与制造执行系统（MES）的数据边界，实现"教学任务驱动生产、生产数据反哺教学"的闭环。

### 1.2 核心设计理念
- **分布式生产，集中式管理**：教师端创建内容直接发布，教务端事后管控
- **教学即生产，生产即教学**：加工类试卷自动转化为生产工单
- **数据驱动评价**：成绩=理论分+工艺分+质量分+效率分
- **资源竞争与共享**：教室/设备/刀具在教学与生产间动态分配

### 1.3 业务背景与痛点
| 痛点 | 解决方案 |
|------|----------|
| 实训脱离实际 | 加工类任务自动生成真实生产工单 |
| 教学评价主观 | 设备数据客观记录，成绩=理论+质量+效率 |
| 资源利用率低 | 教学与生产资源统一调度，动态分配 |
| 产教融合形式化 | 业务流深度融合，任务自动联动 |

## 二、用户角色与权限

### 2.1 角色定义

#### 教务管理员
- **职责**：教学资源统筹、基础数据维护、质量监控、全局分析
- **权限特点**：知识库**只读管理**，无法新增内容，可下架违规内容

#### 教师
- **职责**：教学内容生产、生产任务管理、实训指导、成绩评定
- **权限特点**：知识库**内容生产者**，生产工单管理者

#### 学生
- **职责**：课程学习、实训任务执行、考试考核、错题复习
- **权限特点**：任务接收者，生产执行者

### 2.2 权限矩阵（知识库模块）

| 功能 | 教师端 | 教务端 | 说明 |
|------|--------|--------|------|
| **新增/创建** | ✅ | ❌ | 教师创建，教务端无法新增 |
| **查看/浏览** | ✅ | ✅ | 双方均可查看 |
| **编辑/修改** | ✅（自己创建） | ❌ | 教务端不能修改 |
| **删除** | ✅（自己创建） | ❌ | 仅创建者可删除 |
| **下架** | ✅（自己创建） | ✅（全部） | 教务端可下架违规内容（事后管控） |
| **发布** | ✅ 直接发布 | ❌ | 教师发布即生效，无需审核 |
| **分类/标签** | ✅ | ✅ | 双方均可维护。指知识库内容（视频/教材/题目）的分类目录和标签，用于内容检索和筛选 |
| **统计/分析** | ✅（个人） | ✅（全局） | 教务端查看全局使用情况 |

## 五、核心业务流程

### 5.1 加工类试卷→生产工单自动转化

#### 触发条件
- **触发页面**：教师端-实训任务页、综合试卷管理页
- **触发动作**：点击"发布"按钮
- **必要条件**：
  - `题目类型` = "加工类"
  - **已选择排课记录**（生产任务数据源）
  - `工艺参数` ≠ NULL
  - `设备要求` ≠ NULL

#### 关键业务规则
**所有生产任务必须基于排课数据**：
- 教师发布任务时必须选择**排课记录**
- 自动带出：上课时间、教室、设备、班级学生
- 生产工单的时间/地点/人员全部来自排课

#### 字段映射规则

**核心规则**：加工类任务按学生人数生成对应数量的生产工单，每名学生对应1个工单，每个工单生产数量为1。

| 排课字段 | 生产工单字段 | 说明 |
|---------|-------------|------|
| 排课ID | schedule_id | 核心关联字段（所有工单共用）|
| 上课日期 | schedule_date | 生产日期（固定，不调整）|
| 上课时间段 | schedule_time | 生产时段（固定，不调整）|
| 教室ID | classroom_id | 生产地点（固定）|
| 学生ID | student_id | 关联学生（每工单1人）|
| 设备ID | equipment_id | 生产设备（APS排程后分配）|
| 固定值 | plan_qty | 计划生产数量 = 1（固定）|

#### 流程时序
```
教师创建加工类试卷
    │
    ├── 选择排课记录（必选）
    │   ├─ 自动带出上课时间/教室/设备/班级
    │   └─ 自动带出班级学生列表
    │
    ├── 配置工艺参数/设备要求/质量要求
    │
    ├── 点击"发布"
    │
    └── 【自动执行】
        ├─ 1. 同步到知识库（教务端可见）
        ├─ 2. 推送任务给学生
        │
        └── 【APS排程阶段 - 非实时】
            ├─ 3. 读取调度要素配置（设备效率/转运时间/优先级）
            ├─ 4. 【APS自动排程】
            │   ├─ 为每名学生生成独立生产工单（每单数量=1）
            │   ├─ 分配具体设备（考虑设备数量，按排队逻辑）
            │   └─ 生成工序计划
            ├─ 5. 基于排程结果生成刀具需求计划（结合刀具BOM）
            └─ 6. 学生收到工单分配通知（含设备信息）
```

**重要说明**：
- **时间固定**：APS排程不考虑调整时间，排课的时间/地点固定不变
- **设备分配**：APS仅做设备分配和排队优化（资源不足时按排队逻辑）
- **工单生成时机**：APS排程后生成，不是在发布时立即生成

### 5.2 学生答题与设备数据同步

#### 实训题目四大类型

| 类型 | 答题形式 | 批阅方式 | 特殊说明 |
|------|----------|----------|----------|
| **设计类/编程类** | 填写内容+上传附件 | 人工批阅 | 支持多文件上传 |
| **工艺类** | 基于生产工艺数据填写多道工序参数 | 人工批阅 | 关联产品工艺库 |
| **加工类** | 设备加工+**主动同步设备数据** | 人工批阅 | 数采设备数据由学生点击同步，系统强制要求同步后才能提交 |
| **检测类** | 对检测方案各项进行检测并填写数据 | 人工批阅 | 学生填写检测数据，教师批阅时参考系统提示进行评分 |

#### 加工类答题流程
```
学生接收任务
    │
    ├── 工艺设计阶段
    │   └─ 填写工艺参数（同工艺类）
    │
    ├── 按排课时间到达指定设备（APS分配）
    │
    ├── 设备加工操作
    │
    ├── 工序完成后
    │   │
    │   └── 点击【同步设备数据】按钮◄─── 强制步骤
    │       │
    │       └── 系统实时调用数采接口
    │           ├─ 获取当前工序关联设备数据
    │           ├─ 运行时间/转速/进给/报警/OEE
    │           └─ 数据存入成绩表.equipment_data
    │
    └── 提交加工成果
        │
        └── 【强制校验】未同步设备数据则无法提交
```

**关键规则**：
- 设备数据**非自动采集**，由学生**主动点击同步**获取
- **强制校验**：未点击同步设备数据，系统不允许提交任务
- 学生必须完成设备数据同步后才能提交成果

### 5.3 普通任务批阅机制

| 题型 | 批阅方式 | 说明 |
|------|----------|------|
| 单选题 | **自动批阅** | 匹配预设答案 |
| 多选题 | **自动批阅** | 全对满分，部分对按比例，错选0分 |
| 判断题 | **自动批阅** | 匹配预设答案 |
| 填空题 | **自动批阅** | 支持多答案匹配 |
| 简答题 | **人工批阅** | 教师评分 |
| 论述题 | **人工批阅** | 教师评分 |
| 计算题 | **人工批阅** | 教师按步骤评分 |

**成绩计算**：自动批阅得分 + 人工批阅得分 = 最终得分

**成绩来源**：
- 成绩来源于学生提交的任务/试卷，经任课教师批阅后生成
- 普通任务/考试：客观题自动批阅，主观题教师评分
- 实训任务：教师根据学生提交内容（含同步的设备数据）进行各维度评分
- 缺席学生：教师批阅时记为0分

### 5.4 考试管理（综合组卷）

支持混合组卷：
- 普通题目（单选/多选/判断/填空/简答）
- 实训题目（设计/编程/工艺/加工/检测）

**批阅流程**：
```
学生提交试卷
    │
    ├── 自动批阅部分（立即计算得分）
    │
    ├── 人工批阅部分（进入教师批阅队列）
    │       │
    │       └── 教师逐题批阅
    │           ├─ 简答题：查看答案，输入分数
    │           ├─ 设计类：查看内容+附件，评分
    │           ├─ 工艺类：查看工艺参数，评分
    │           ├─ 加工类：查看同步数据+成果，评分
    │           └─ 检测类：查看检测结果，复核
    │
    └── 汇总成绩（自动+人工）
```

- 生产关联：关联工单数、工单完成率
- 排课关联：上课时间、教室、设备
- **操作**：查看详情、下架

### 6.2 教师端-APS智能排程（生产工单管理）

#### 6.2.1 系统定位
APS（Advanced Planning and Scheduling）模块是连接学生工艺编制与车间实训任务安排的核心调度引擎，实现教学任务的智能排程优化。

#### 6.2.2 核心业务流程

```
学生编制工艺（如需要）
    │
    ├── 设定调度要素（设备效率/转运时间/维保周期/物料时间/优先级）
    │
    ├── 设定工作日历（正常上课时间/非正常时间/节假日）
    │
    ├── 【系统自动排程】
    │   │
    │   ├── 定时自动排程（每日一次或多次，最高0.5小时/次）
    │   ├── 产能负载计算（预测瓶颈资源）
    │   ├── 任务均衡（避免设备忙闲不均）
    │   ├── 成套优先（同一订单零件集中完成）
    │   └── 动态调整（根据前道进度自动调整后道）
    │
    ├── 排程结果检查
    │   ├─ 正常 → 发布实训计划
    │   ├─ 延期 → 显示延期预警（红色高亮+原因分析）
    │   └─ 需调整 → 人工干预调整后重新排程
    │
    └── 执行实训任务
        │
        ├── 实时进度采集
        │
        ├── 异常检测
        │   ├─ 设备故障 → 任务转移至其他设备 → 重新排程
        │   ├─ 紧急插单 → 评估影响 → 重新排程
        │   └─ 返工 → 重新安排返工工序
        │
        └── 动态调整后续计划
```

#### 6.2.3 调度要素设定

| 调度要素 | 功能说明 | 业务规则 |
|---------|---------|---------|
| **设备效率设定** | 根据设备新旧效能设定效率值 | 新设备100%，旧设备按实际折减 |
| **零件转运时间设定** | 设定工序间物料转运时间 | 根据车间布局设定区域间转运时间 |
| **设备维保周期设定** | 设定设备保养及维修周期 | 维保期间不安排生产任务 |
| **物料到达时间设定** | 设定实验物料的计划到厂时间 | 物料未到不启动相关工序排程 |
| **多层级优先级设定** | 支持项目/产品/零件/工序多层级优先级 | 至少支持30级以上优先级区分 |

#### 6.2.4 自动排程功能

| 功能项 | 功能描述 | 业务规则 |
|-------|---------|---------|
| **定时自动排程** | 系统自动执行排程运算 | 每日一次或多次，频次最高0.5小时/次 |
| **工作日历设定** | 设定正常及非正常工作时间 | 支持正常上课时间、非正常时间分别设定 |
| **智能假期识别** | 自动识别放假时间段 | 放假时间段内不予排程 |
| **产能负载计算** | 根据在制零件自动计算 | 预测瓶颈资源，提供产能分析 |
| **任务均衡** | 自动均衡同类设备计划任务 | 避免设备忙闲不均 |
| **投产日期设定** | 支持实验零件的投产日期设定 | 优先安排正常上课时间，不满足时安排非正常时间 |
| **动态调整** | 根据前道任务进度自动调整 | 自动根据实际进度调整后道排产计划 |
| **成套优先** | 同一实验订单优先成套 | 同一订单下同一学生的多道工序集中在相邻时段完成 |

#### 6.2.5 排程规则与约束

**排程规则**：

| 规则编号 | 规则名称 | 规则内容 |
|---------|---------|---------|
| APS-R01 | 优先级规则 | 支持多级优先级设置（≥30级） |
| APS-R02 | 时间规则 | 优先安排正常上课时间，紧急情况安排非正常时间 |
| APS-R03 | 设备约束 | 排程必须考虑设备效率，新旧设备效能不同 |
| APS-R04 | 成套规则 | 同一订单零件优先成套完成 |
| APS-R05 | 动态调整规则 | 根据实际进度自动调整后续计划 |
| APS-R06 | 放假规则 | 放假时间段内不予排程 |
| APS-R07 | 维保规则 | 设备维保期间不安排任务 |
| APS-R08 | 物料规则 | 物料未到达不启动相关工序 |

**约束条件**：
- **工艺约束**：工序间加工顺序和约束关系
- **设备约束**：设备能力、效率、状态
- **时间约束**：工作日历、交期要求
- **物料约束**：物料到达时间
- **人力约束**：操作人员技能、数量

#### 6.2.6 可视化排程界面

**展示内容**：
- **甘特图**：直观展示各工序的时间安排和设备分配
- **设备负载图**：显示各设备的负载情况和利用率
- **工序时间轴**：显示最迟开工时间、剩余缓冲时间

#### 6.2.7 异常处理与重排程

| 异常类型 | 处理逻辑 |
|---------|---------|
| **生产插单** | 支持紧急插单，系统自动重新排程，评估对现有计划的影响 |
| **急加件处理** | 支持紧急加件，优先安排紧急零件，调整其他任务 |

#### 6.2.8 工单列表

## 九、附录

### 9.1 术语表
| 术语 | 说明 |
|------|------|
| 加工类题目 | 包含实际加工操作的实训题目，会触发生产工单 |
| 排课 | 课程安排，包含时间/地点/设备/学生，是生产任务的数据源 |
| 同步设备数据 | 学生主动点击按钮获取设备实时数据的操作 |
| OEE | 设备综合效率（Overall Equipment Effectiveness）|
| 自动生单 | 加工类试卷发布时自动生成生产工单的机制 |
| **APS** | **高级计划与排程（Advanced Planning & Scheduling）** |
| **智能排程** | 基于多约束条件自动生成最优生产计划的过程 |
| **甘特图** | 可视化展示任务时间安排和进度的图表 |
| **设备负载** | 设备在特定时间段内的工作饱和度 |
| **瓶颈资源** | 限制产能的关键设备或工序 |
| **成套优先** | 同一订单的零件优先集中完成的排程策略 |
| **动态调整** | 根据实际执行进度自动调整后续计划的能力 |
| **延期预警** | 系统自动识别并提示可能延期的任务 |
| **缓冲时间** | 工序计划时间与最迟完成时间之间的余量 |

### 9.2 核心业务流程图

#### 9.2.1 加工类试卷→生产工单→APS排程全流程

```
教师发布加工类任务/试卷
    │
    ├── 选择排课（确定时间/地点/班级）
    │   └─ 自动带出班级学生列表
    │
    ├── 选择参与学生（默认全班，可剔除缺席学生）
    │
    ├── 配置工艺参数/设备要求/质量要求
    │
    ├── 点击"发布"
    │
    └── 【阶段1：任务发布】
        ├─ 1. 同步到知识库（教务端可见）
        └─ 2. 推送任务给学生
            │
            └── 【阶段2：APS智能排程】（定时执行，非实时）
                │
                ├── 3. 读取调度要素配置
                │   ├─ 设备效率设定
                │   ├─ 转运时间设定
                │   ├─ 维保周期设定
                │   ├─ 物料到达时间
                │   └─ 多层级优先级设定
                │
                ├── 4. 【APS自动排程】
                │   ├─ 为每名学生生成独立生产工单（每单数量=1）
                │   ├─ 分配具体设备（时间/地点不变，仅分配设备）
                │   ├─ 资源不足时按排队逻辑处理
                │   └─ 同一学生的多道工序成套排程
                │
                ├── 5. 生成排程结果（甘特图/设备负载图）
                │
                ├── 6. 基于排程结果生成刀具需求计划（结合刀具BOM）
                │   └─ 若排程调整，刀具需求重新计算
                │
                └── 7. 推送工单分配通知给学生（含设备信息）
                    │
                    └── 【阶段3：学生执行】
                        │
                        ├── 学生按排课时间到达指定设备
                        │
                        ├── 设备加工操作
                        │
                        ├── 【强制】点击同步设备数据
                        │   ├─ 获取运行时间/转速/进给/OEE等
                        │   └─ 未同步则无法提交
                        │
                        ├── 异常处理（设备故障）
                        │   └─ 临时分配其他设备，原工单作废
                        │
                        └── 提交加工成果
                            │
                            └── 【阶段4：教师批阅】
                                │
                                ├── 查看同步的设备数据
                                ├── 查看加工成果
                                ├── 各维度评分（理论/工艺/质量/效率）
                                ├── 成绩汇总
                                └── 批阅结束后自动生成错题集
```
