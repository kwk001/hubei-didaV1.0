## 方案概述

### 1.1 设计目标

实现机场行李全流程的自动化识别与追踪，通过RFID技术实现：

- **实时性**: 行李状态秒级更新
- **准确性**: 识别准确率 ≥ 99.9%
- **可追溯**: 全程轨迹完整记录
- **高并发**: 支持每秒1000+行李识别

### 1.2 应用场景

| 场景 | 位置 | 功能 |
|-----|------|------|
| 值机托运 | 值机柜台 | 行李标签打印与RFID写入 |
| 安检通道 | 安检区域 | 行李自动识别与安检关联 |
| 分拣系统 | 分拣区域 | 自动分拣路径识别 |
| 装载区域 | 停机坪 | 行李装载确认 |
| 卸载区域 | 到达厅 | 行李到达确认 |
| 提取转盘 | 到达厅 | 行李提取确认 |

## 接口协议

### 6.1 设备通信协议

#### 6.1.1 LLRP协议

用于与固定式RFID读写器通信的标准协议。

```java
// 创建LLRP连接
LLRPConnection conn = new LLRPConnection();
conn.setHost("192.168.1.101");
conn.setPort(5084);
conn.connect();

// 配置读取参数
ROSpec roSpec = new ROSpec();
roSpec.setROSpecID(1);
roSpec.setPriority(0);
roSpec.setCurrentState(ROSpecState.Disabled);

// 设置Inventory参数
InventoryParameterSpec inventoryParam = new InventoryParameterSpec();
inventoryParam.setInventoryParameterSpecID(1);
inventoryParam.setProtocolID(AirProtocols.EPCGlobalClass1Gen2);
roSpec.addToInventoryParameterSpecList(inventoryParam);
```

#### 6.1.2 MQTT协议

用于设备状态上报和远程控制。

**主题定义**:

| 主题 | 方向 | 说明 |
|-----|------|------|
| `rfid/{deviceId}/status` | 设备→平台 | 设备状态上报 |
| `rfid/{deviceId}/data` | 设备→平台 | 标签数据上报 |
| `rfid/{deviceId}/cmd` | 平台→设备 | 控制命令下发 |
| `rfid/{deviceId}/ack` | 设备→平台 | 命令响应 |

**状态上报消息**:

```json
{
  "deviceId": "RF001",
  "timestamp": "2026-03-12T10:15:00Z",
  "status": "ONLINE",
  "metrics": {
    "cpuUsage": 45,
    "memoryUsage": 60,
    "temperature": 42,
    "readCount": 12345,
    "errorCount": 0
  }
}
```
